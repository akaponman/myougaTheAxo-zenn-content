## マルチエージェント構成

Claude Codeは`Task`ツールで複数のサブエージェントを同時に動かせます。それぞれが独立してコードを読み、分析し、変更を加えます。

## 利用可能なエージェントタイプ

| タイプ | ツール | 主な用途 |
|-------|-------|---------|
| `general-purpose` | 全ツール | 実装、リファクタリング |
| `Explore` | 読み取り系のみ | コード調査、パターン検索 |
| `Plan` | 読み取り系のみ | 設計・計画立案 |
| `Bash` | Bashのみ | コマンド実行特化 |

## パターン1: 調査と実装の並列化

```
ユーザー: 「認証システムを追加して」

メインエージェント:
  ├── Explore Agent: 既存の認証コードを調査
  ├── Explore Agent: 使用ライブラリのドキュメント確認
  └── (結果を統合して実装計画を立案)
      └── general-purpose Agent: 実装実行
```

**ポイント:** `Explore`は読み取り専用で高速。調査フェーズを並列化することで、メインエージェントのコンテキストを汚さずに情報収集できます。

## パターン2: テスト駆動開発フロー

1. `Plan`エージェントで実装計画を策定
2. ユーザーが計画を承認
3. `general-purpose`エージェントが実装
4. テスト実行で品質確認

## パターン3: チーム分業（Team機能）

大規模タスクでは`Team`を構成し、タスクリストで進捗管理できます：

```
Team Lead (メインエージェント)
  ├── frontend-dev: UIコンポーネント実装
  ├── backend-dev: API実装
  └── tester: テスト作成・実行
```

各メンバーはタスクリストを共有し、完了報告やブロッカー通知を行います。

## 実践的な使い方

```
「APIエンドポイント3つとフロントのフォーム2つを追加して」

→ バックエンドとフロントエンドを別エージェントに振り分け
→ 両方完了後にインテグレーション確認
→ 全体でテスト実行
```
