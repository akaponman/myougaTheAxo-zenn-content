## ユースケース1: レガシーコードのリファクタリング

CLAUDE.mdに追記:
```markdown
## リファクタリング方針
- 外部インターフェースは変更しない
- テストが通ることを各ステップで確認
- 1コミット = 1論理的変更
- 変更理由をコミットメッセージに記載
```

## ユースケース3: バグ修正ワークフロー

```
「Issue #123 を修正して」

→ Explore Agent: 関連コードを調査
→ 原因の仮説を提示
→ 修正コードを生成
→ テストで確認
→ /commit でコミット
```

## ユースケース4: セッション管理のコツ

```bash
# 前回のセッションを再開
claude --resume

# 非対話モード（CI/スクリプト用）
claude --print "このファイルのバグを教えて" src/main.py

# コスト管理（最大ターン数制限）
claude --max-turns 10
```

## ユースケース5: 効率的な日常ワークフロー

```
1. プロジェクト開始時
   $ claude
   → CLAUDE.mdが自動読み込み、コンテキスト理解済み

2. 作業中
   /compact → コンテキストを圧縮して継続
   /clear → 新しい話題に切り替え

3. コミット時
   /commit → 変更内容を分析して適切なメッセージ生成

4. レビュー時
   /review-pr → PR差分を自動レビュー（カスタムスキル）
```
